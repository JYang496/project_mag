# 项目美术设计指南（基于当前项目结构）

## 1. 你这个项目当前的视觉约束（从代码与场景反推）

- 玩家使用 8 向贴图（`asset/images/characters/*.png`），并在 `Player/Mechas/scripts/Player.gd` 中根据瞄准角度切换方向贴图。
- 武器会围绕玩家持续显示并轨道运动（`Player/Mechas/scripts/Player.gd`），因此武器在小尺寸下必须保持高辨识度轮廓。
- 战场是 NxN 格子，格子归属通过颜色调制区分（玩家/敌人/中立，见 `Board/Cells/cell.gd`），所以格子底图要“可被染色”（中性、不过饱和）。
- UI 包含库存/商店/升级/模块等重系统（`UI/scenes/UI.tscn`），图标一致性的重要性不低于角色本体美术。
- 当前世界为较大俯视区域（`World/board_cell_generator.gd` 中 `cell_spacing = Vector2(510,510)`），环境细节不能太噪，否则会影响战斗可读性。

## 2. 推荐整体风格方向

- 主题：`废土机甲竞技场（scrap-tech mecha arena）`，工业感 + 能量科技感。
- 俯视可读性优先：每个精灵保持清晰外轮廓、2-3 个明度层级、单体少色高对比。
- 阵营配色语言：
- 玩家/机甲：青蓝 + 白色高光。
- 敌人：红/橙高亮。
- 中立环境：低饱和钢铁灰/石色。
- 掉落与奖励：金币金黄、芯片青绿、稀有模块紫色。

## 3. 从 0 开始的资产生产计划

### Phase A（先做可玩垂切）

- 1 套机甲基础 8 方向。
- 3 种敌人（近战小怪、远程小怪、精英）。
- 4 把武器外观（手枪、霰弹、激光、火箭）。
- 6 组核心弹道/命中特效（开火火花、子弹拖尾、命中火花、爆炸、激光束、拾取爆点）。
- 1 套生物群系地表 Tileset + 1 套格子状态覆盖层（中立/玩家/敌人/争夺/锁定）。
- 1 套 UI 基础件：槽位边框状态、按钮状态、20 个核心武器/模块/道具图标。

### Phase B（内容扩展）

- 完成全部机甲职业视觉（Prototype / Ranger / Melee / Collector / Turret）。
- 对齐 `data/weapons.json` 的全武器图标与场景内武器外观。
- 敌人扩展与精英变体。
- 商人/铁匠 NPC 立绘或场景精灵。

### Phase C（打磨）

- 伤害数字样式统一。
- 阶段切换（Prepare/Battle/Reward）视觉过场。
- Boss 登场卡面（如果后续加 Boss）。
- 稀有度进阶特效、环境氛围特效（尘埃、微光、热浪等）。

## 4. Godot 落地的技术美术规则

- 固定统一世界比例并严格执行（你代码里玩家贴图会归一到 `48x48`，角色设计应围绕这个占地级别）。
- 角色与敌人 pivot 建议统一在“底盘中心/脚底中心”，保证旋转、受击、位移视觉稳定。
- 分层出图：底色层 / 发光层 / 可选描边层，便于换色与稀有度改色。
- 图标统一 atlas 规格（你的 UI 槽位很多，统一尺寸能显著降成本）。
- 优先用 shader/tint 做阵营色和稀有度变化，减少重复重绘。

## 5. 动画与特效优先级

- 机甲：待机悬浮（轻微）、开火后坐、受击闪白、死亡爆点。
- 武器：开火帧、换弹脉冲、过载（overcharge）特征特效。
- 敌人：攻击前摇必须清晰（telegraph），死亡反馈要“脆”并明确掉落。
- 格子玩法可视化：被争夺/被占领时的边缘呼吸光效（轻量即可）。

## 6. UI 视觉设计方向

- Shop / Upgrade / Gear Fuse / Inventory / Module 使用同一套面板语言（避免像多个游戏拼接）。
- 槽位边框清晰表达稀有度（普通/优秀/稀有/史诗/传说）。
- 图标先做轮廓区分，再做颜色区分，保证色盲与高压战斗场景可读。
- 文案是辅助，信息应主要靠图标、边框态、数字层级表达。
