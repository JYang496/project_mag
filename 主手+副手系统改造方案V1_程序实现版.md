# 主手+副手系统改造方案（V1）- 程序实现版

## 1. 目标与边界

- 目标：从“多武器齐射”改为“1主武器主动输出 + 副手被动触发”。
- 边界：玩家持有1-4把武器；任意时刻仅1把主武器；副手不主动发射攻击体。
- 非目标：本版本不重做全部武器技能美术，仅定义可扩展框架与最小可行实现。

---

## 2. 系统架构（模块拆分）

### 2.1 核心模块
1. `WeaponLoadoutService`：管理玩家持有武器、主副手标记、切换逻辑。  
2. `MainWeaponController`：处理主武器普通攻击与主动技能释放。  
3. `OffhandPassiveController`：管理副手被动注册、触发、ICD与互斥。  
4. `WeaponSwapController`：处理切换CD、退场冷却、全局锁、接力奖励。  
5. `CombatEventBus`：统一分发命中、击杀、状态变化等事件。  
6. `WeaponHUDPresenter`：主副手UI、CD、资源锁、失败原因展示。

### 2.2 数据层
1. `WeaponRuntimeState`：武器运行时状态（CD、锁定、资源快照）。  
2. `MainBonusConfig`：主手基础加成+专精加成配置。  
3. `ActiveSkillConfig`：主动技能配置。  
4. `OffhandPassiveConfig`：副手被动配置。  
5. `SwapRuleConfig`：切换规则参数。  
6. `TriggerRuleConfig`：触发条件配置（标签、事件、阈值）。

## 4. 状态机设计（伪代码）

## 4.1 顶层战斗状态机
```pseudo
state PlayerCombatState {
  Idle,
  MainAttacking,
  MainActiveCasting,
  SwapPending,
  SwapLocked
}

on Input.AttackPressed:
  if current_state in [SwapPending, SwapLocked]: reject(FailReason.SWAP_LOCKED)
  else MainWeaponController.TryNormalAttack()

on Input.ActiveSkillPressed:
  MainWeaponController.TryCastActive()

on Input.SwapMainPressed(target_slot):
  WeaponSwapController.TrySwapMain(target_slot)
```

## 4.2 主武器普通攻击
```pseudo
function TryNormalAttack():
  main = Loadout.GetMainWeapon()
  if main.normal_attack_cd_remaining > 0:
    return Fail(CD_LOCKED)

  if not ResourceService.CanPayNormalAttack(main):
    return Fail(RESOURCE_LACK)

  ApplyMainBonus(main)
  SpawnMainAttack(main)                  // 只主武器发射
  PublishEvent(ON_MAIN_ATTACK_FIRED)

  main.normal_attack_cd_remaining = CalcAttackCD(main)
  OffhandPassiveController.Evaluate(ON_HIT / ON_MAIN_ATTACK_FIRED)
```

## 4.3 主武器主动技能
```pseudo
function TryCastActive():
  main = Loadout.GetMainWeapon()
  cfg = ActiveSkillConfig[main.weapon_id]

  if main.active_skill_cd_remaining > 0:
    return Fail(CD_LOCKED)

  if not ResourceService.CanPay(main, cfg.resource_cost_type, cfg.resource_cost_value):
    return Fail(RESOURCE_LACK)

  EnterState(MainActiveCasting)
  wait cfg.cast_frontswing_sec

  if cfg.interruptible and Player.IsControlled():
    refund = cfg.cooldown_sec * cfg.interrupted_cd_refund_ratio
    main.active_skill_cd_remaining = cfg.cooldown_sec - refund
    EnterState(Idle)
    return Fail(CHANNEL_INTERRUPTED)

  damage_mult = ResolveTriggerWindowBonus(main, cfg)
  ExecuteActiveSkill(main, damage_mult)
  ResourceService.Pay(main, cfg.resource_cost_type, cfg.resource_cost_value)

  main.active_skill_cd_remaining = cfg.cooldown_sec
  WeaponSwapController.ApplyGlobalSwapLockAfterActive()
  PublishEvent(ON_MAIN_ACTIVE_CAST)
  EnterState(Idle)
```

## 4.4 副手被动触发器
```pseudo
function Evaluate(event_type):
  offhands = Loadout.GetOffhands()
  context = CombatContext.Build(event_type)

  for p in PassiveRegistry.GetPassives(offhands):
    if p.trigger_event != event_type:
      continue

    if ICDBlocked(p):
      HUD.NotifyPassiveFail(p, ICD_LOCKED)
      continue

    if MutexBlocked(p):
      HUD.NotifyPassiveFail(p, MUTEX_BLOCKED)
      continue

    if not CheckCondition(p.trigger_rule.condition_expr, context):
      HUD.NotifyPassiveFail(p, CONDITION_NOT_MET)
      continue

    if Random() > p.trigger_rule.chance:
      continue

    ApplyPassiveEffect(p, context)
    StartICD(p)
    MarkMutex(p.mutex_group)
    PublishEvent(OFFHAND_PASSIVE_TRIGGERED)
```

## 4.5 主武器切换
```pseudo
function TrySwapMain(target_slot):
  if global_swap_cd_remaining > 0:
    return Fail(SWAP_LOCKED)

  if global_swap_lock_remaining > 0:
    return Fail(SWAP_LOCKED)

  target = Loadout.GetWeaponBySlot(target_slot)
  if target.reenter_lock_remaining > 0:
    return Fail(SWAP_LOCKED)

  old_main = Loadout.GetMainWeapon()
  old_main.role = OFFHAND
  old_main.reenter_lock_remaining = SwapRule.reenter_lock_sec
  RemoveMainBonus(old_main)

  target.role = MAIN
  ApplyMainBonus(target)

  global_swap_cd_remaining = SwapRule.swap_cd_sec
  PublishEvent(ON_SWAP_TO_MAIN)

  TryApplyRelayBonus(old_main, target)
  HUD.NotifyMainSwapped(old_main, target)
```

## 5. 事件流（简化）

```text
Input -> MainWeaponController -> CombatEventBus
      -> OffhandPassiveController -> EffectApply
      -> DamageSystem -> CombatEventBus (OnHit/OnKill)
      -> HUDPresenter (状态更新/失败原因)
```

关键事件建议：
- `ON_MAIN_ATTACK_FIRED`
- `ON_MAIN_ACTIVE_CAST`
- `ON_HIT`
- `ON_KILL`
- `ON_SWAP_TO_MAIN`
- `OFFHAND_PASSIVE_TRIGGERED`
- `ACTION_FAILED(reason)`

## 8. 存档与回放兼容

需要新增存档字段：
- 当前主武器slot id
- 每把武器的主动技能CD剩余
- 每把武器reenter lock剩余
- 副手被动ICD快照（可选，若不存则读档重置）

兼容策略：
- 旧存档加载时，默认slot0为Main；其余为Offhand。
- 缺失字段按0初始化。

## 9. 监控埋点与指标

必埋点：
1. `main_swap_count`（每局）
2. `active_skill_cast_count`（每分钟）
3. `action_fail_reason_dist`
4. `offhand_passive_trigger_count_by_id`
5. `dps_contribution_main_vs_offhand`

目标区间：
- Main输出占比：55%~70%
- Offhand输出占比：30%~45%
- 单副手贡献：<=15%
- 主动技能贡献：20%~30%

## 10. 集成顺序（程序落地）

1. 先接入`WeaponLoadoutService`与`WeaponRole`，禁用Offhand主动发射。  
2. 上`MainBonusConfig`，确保切主手有即时收益。  
3. 接`ActiveSkillConfig`与主技能CD。  
4. 接`WeaponSwapController`（切换CD、退场冷却、全局锁）。  
5. 接`OffhandPassiveController`（先做1类被动）。  
6. 接HUD最小集（主副手、CD、失败原因）。  
7. 最后接敌人反制与波次规则。

## 11. MVP代码验收清单

1. 攻击键触发时，仅Main发射攻击体（自动化测试）。  
2. Offhand被动能触发且遵守ICD与互斥。  
3. 主动技能受CD/资源/打断约束，失败原因可上报。  
4. 切换主武器受切换CD与退场冷却限制。  
5. HUD可稳定显示：当前Main、主动技能CD、资源锁原因、最近触发日志。  
6. 指标上报完整，能计算Main/Offhand输出占比。
