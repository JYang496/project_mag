# MagArena 项目概览

更新日期：2026-07-15。本文只描述当前高层运行结构；具体规则以代码、资源和 `docs/` 中标为当前的合同为准。

## 核心循环

1. 玩家进入准备区，使用购买、升级、仓库和棋盘服务调整构筑。
2. 在中心区域长按开始交互，完成临时模组和棋盘确认。
3. 非 Boss 关从两个不同的战斗协议中选择一个；Boss 关直接沿用关卡规则。
4. 战斗中击杀敌人、完成协议主目标和可选格子任务，并收集金币与经验。
5. 协议完成后进入统一胜利转场、奖励草案和下一轮准备阶段；生命归零进入 Game Over。

## 战斗协议

- 歼灭：有限生成预算，按批次释放，预算耗尽且场上无敌人时完成。
- 坚守：持续生成并逐步提高威胁，坚持 45/55/65 秒后撤离残敌。
- 行动：依次占领两个地面信标，离开范围暂停、敌人在范围内降低推进速度。
- 协议领域通过 `BattleContractCombatPort` 与当前 `EnemySpawner` 解耦。
- 选择历史、战前选项和已选协议参与持久化及战斗回滚。

稳定边界见 [`docs/plans/battle_contract_combat_port.md`](docs/plans/battle_contract_combat_port.md)。

## 世界与表现架构

- 玩法权威仍是 2D：移动、碰撞、AI、伤害、生成和区域检测均使用 2D 节点。
- 地板、格子边界、范围提示和信标通过透视 `Camera3D` 显示为地面 Mesh。
- 角色、敌人、武器和掉落物使用投影/Billboard 表现。
- HUD 和菜单位于 `CanvasLayer`，不参与世界投影。

架构说明见 [`docs/plans/hybrid_2d_3d_2_5d_architecture.md`](docs/plans/hybrid_2d_3d_2_5d_architecture.md)。

## 主要系统

- 玩家：`Player/Mechas/scripts/Player.gd` 与独立移动、武器、技能和拾取运行组件。
- 武器：远程/近战基础框架、升级、融合、分支和武器模组。
- 生成：`LevelCombatPlan`、HP 预算、压力释放、存活上限和出生点选择。
- 棋盘：3×3 格子、active cell、地形效果、任务模组和右侧次目标 HUD。
- 奖励：标准战后草案与格子任务奖励分离；协议表现奖励不改变基础奖励池。
- 设置：分辨率、本地化、控制提示和混合摄像机调试能力。

## 存档与回滚

- `SaveManager` 统一管理版本化槽位存档、主备文件、原子替换和继续游戏恢复。
- 玩家、装备、格子效果、任务模组、奖励草案与协议历史统一写入 `user://saves/slot_0/run.json`；旧运行时 JSON 仅作为兼容缓存，不是 Continue 的权威来源。
- 战前回滚快照恢复到准备阶段，不恢复半场敌人、计时或信标进度。
- 主存档只在战斗胜利流程中提交：返回准备阶段先保存，战后奖励成功领取后更新同一胜利存档；战斗中退出或 Game Over 后继续会回到最近一次已结算胜利后的准备阶段。
- 新游戏会清理旧槽位、协议历史和相关运行时状态。

## 操作入口

- `WASD` / 方向键：移动。
- 鼠标左键：攻击和 UI 操作。
- `F`：交互。
- `Q` / `E`：切换武器。
- `Space`：玩家主动技能。
- `R`：武器技能。
- `Esc` / 右键：暂停、返回或取消当前模态操作。

## 验证

测试布局和命令以 [`tests/README.md`](tests/README.md) 为准。当前没有注册的自动玩法测试；修改后至少运行 Godot `--check-only`，并对受影响流程进行编辑器内人工验证。
