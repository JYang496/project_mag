# 主手+副手系统改造方案（V1）

## 实施路线图

### Phase 1（2-3周，系统骨架，P0）
1. M1 主手/副手槽位与运行时状态机  
2. M2 主武器性能加成（基础+专精）  
3. M3 主动技能系统（CD/资源/前摇打断）  
4. M4 副手被动系统（分类/叠加/互斥/频率上限）  
5. M5 主武器切换限制（切换CD/退场冷却/全局锁/接力奖励）  
6. M8 HUD基础反馈（主副手、CD、资源锁）

### Phase 2（2周，战斗解题与可读性，P1）
1. M6 解题型敌人（护盾怪/冲锋怪/召唤怪）  
2. M7 波次编排规则与关卡挂钩  
3. M9 触发反馈与战斗日志（触发成功/失败原因）

### Phase 3（1-2周，数值闭环，P1-P2）
1. M10 预算与平衡框架（占比、上限、指标、AB参数）  
2. 根据测试回收调参（CD、触发率、敌人效率惩罚）

---

## 改造清单明细

## M1 主手+副手状态机与槽位重构
- 目标问题：所有武器同按键输出，缺少角色内战术分工。
- 具体改动：  
  - 规则：玩家持有1-4武器；任意时刻仅1把`Main`，其余`Offhand`。  
  - 状态机：`Idle -> MainAttack -> MainSkillCast -> SwapPending -> SwapLock`。  
  - 触发条件：普通攻击仅驱动Main；Offhand只运行被动监听器。  
  - 限制：无武器时禁用攻击输入；1把武器时自动Main且禁切换。
- 涉及模块：战斗逻辑、输入层、武器管理器、存档。
- 对玩法影响：  
  - 收益：明确“操作主语”，提升操作意图。  
  - 代价：旧构筑会感觉输出下降，需要补偿主手增益。
- 风险与规避：  
  - 风险：前期手感“变弱”。  
  - 规避：上线初期主手基础增益先给高值（如伤害+25%）再回调。
- 验收标准：  
  - 攻击键触发事件中，Main贡献事件占比=100%。  
  - Offhand主动发射事件占比=0%。
- 优先级：P0  
- 预计工作量：中

## M2 主武器性能加成（基础加成+专精）
- 目标问题：主手身份没有收益，切换缺少动力。
- 具体改动：  
  - 基础加成区间：伤害`+15%~25%`，攻击频率`+10%~20%`，投射速度`+10%~30%`。  
  - 专精加成：按武器标签给1条（例：Heat类“过热阈值+10%~20%”，Freeze类“异常积累+15%”）。  
  - 生效规则：切为Main后`0.1s`生效，离开Main立即失效。
- 涉及模块：数值表、武器标签系统、buff系统、UI图标。
- 对玩法影响：  
  - 收益：鼓励围绕Main构筑。  
  - 代价：副手存在感下降，需要被动系统补位。
- 风险与规避：  
  - 风险：个别武器吃双重乘区过强。  
  - 规避：专精采用“加法区”优先，不与核心乘区叠乘。
- 验收标准：  
  - Main切换前后，训练木桩10秒DPS差异稳定在`+18%~35%`。
- 优先级：P0  
- 预计工作量：中

## M3 主动技能系统（内置CD+资源+前摇/打断）
- 目标问题：主手仅“平A增强”，缺少爆发时机决策。
- 具体改动：  
  - 每武器1个主动技能槽。  
  - CD区间：`12~25秒`；强控/无敌类`18~30秒`。  
  - 资源消耗：热量/能量/连击点之一；建议消耗主资源`20%~40%`。  
  - 前摇：`0.15~0.45秒`；受控可打断，打断返还`30%~50%`CD。  
  - 触发窗口：可选“命中X次强化主动技”（X建议`3~8`）。
- 涉及模块：技能系统、资源系统、动画状态、输入缓冲、CD管理。
- 对玩法影响：  
  - 收益：形成爆发窗口与战术保留。  
  - 代价：操作门槛上升。
- 风险与规避：  
  - 风险：主动技能成为唯一解。  
  - 规避：主动技能贡献控制在总输出`20%~30%`。
- 验收标准：  
  - 主动技能使用率（每分钟）达到`1.5~4.0次`。  
  - 失败释放原因可统计（资源不足/冷却/打断）。
- 优先级：P0  
- 预计工作量：大

## M4 副手被动系统（分类/叠加/互斥/触发频率）
- 目标问题：副手价值模糊，容易重新变成隐性齐射。
- 具体改动：  
  - 被动分类：输出修正、功能控制、生存防护。  
  - 叠加规则：同类强效果“取高不叠”；弱效果可叠但总上限`+30%`。  
  - 触发频率：每个副手被动ICD `0.4~1.2秒`。  
  - 互斥组：同标签关键被动互斥（如“双重连锁核心”不可同开）。  
  - 单副手上限：单副手折算输出贡献不超过总DPS`15%`。
- 涉及模块：被动框架、标签系统、事件总线、数值表。
- 对玩法影响：  
  - 收益：构筑清晰，副手可选性提升。  
  - 代价：需要更多说明文本。
- 风险与规避：  
  - 风险：隐藏触发导致“玄学”。  
  - 规避：全部被动必须提供可见触发提示与ICD读条。
- 验收标准：  
  - 三副手合计贡献控制在总输出`30%~45%`。
- 优先级：P0  
- 预计工作量：大

## M5 主武器切换机制（限制与奖励）
- 目标问题：无限切主手可能连爆滥用；不切又失去策略价值。
- 具体改动：  
  - 切换CD：`1.5~3.0秒`。  
  - 退场冷却：离开Main后`3~6秒`不可再次成为Main。  
  - 全局锁：主动技能释放后`0.5~1.0秒`禁止切换。  
  - 接力奖励：在`2秒`内完成特定标签接力（如Fire->Freeze）触发`+10%~20%`短增益，持续`2~4秒`。
- 涉及模块：输入、武器管理、buff系统、UI。
- 对玩法影响：  
  - 收益：切换成为“战术动作”，非手速漏洞。  
  - 代价：高频切换玩家需适应。
- 风险与规避：  
  - 风险：限制过强导致“懒得切”。  
  - 规避：先低门槛（CD 1.5s）上线，再看切换频率调高。
- 验收标准：  
  - 平均切换频率`0.2~0.8次/10秒`。  
  - 连续2秒内多次切换的异常行为下降`>80%`。
- 优先级：P0  
- 预计工作量：中

## M6 敌人反制机制（3类解题敌人）
- 目标问题：同一输出循环可通吃全部敌人。
- 具体改动：  
  - 护盾怪：远程减伤`50%~70%`，近战破盾倍率`+100%`。  
  - 冲锋怪：受减速/冻结时冲锋中断，眩晕`0.4~0.8秒`。  
  - 召唤怪：本体低血，持续召唤；AOE命中召唤单位返还主资源`5%~10%`。  
  - 原则：不做硬免疫，只做效率差（错误解法效率保留`30%~50%`）。
- 涉及模块：敌人AI、状态系统、掉落/波次配置、战斗日志。
- 对玩法影响：  
  - 收益：逼迫武器顺序与主手选择。  
  - 代价：理解成本增加。
- 风险与规避：  
  - 风险：玩家误解为“强行克制”。  
  - 规避：弱点UI显式提示+首次教学关。
- 验收标准：  
  - 对应敌人场景中，正确解法击杀时长比错误解法快`1.4~2.2倍`。
- 优先级：P1  
- 预计工作量：中

## M7 波次编排规则
- 目标问题：即使有反制敌人，若不混编仍可单套路碾压。
- 具体改动：  
  - 每波至少混入2类解题敌人。  
  - 每3波至少1波“切主手压力波”（高护盾+高机动混编）。  
  - Boss前两波放“机制预演怪”。
- 涉及模块：关卡生成、刷怪表、难度曲线。
- 对玩法影响：  
  - 收益：促使切换与技能保留。  
  - 代价：关卡设计复杂度上升。
- 风险与规避：  
  - 风险：混编过多导致读不懂。  
  - 规避：单波机制点不超过2个。
- 验收标准：  
  - 玩家在中后期关卡的主手切换次数较旧版提升`>30%`。
- 优先级：P1  
- 预计工作量：中

## M8 UI基础层（主副手/冷却/资源锁）
- 目标问题：系统复杂后无反馈会被感知为随机。
- 具体改动：  
  - 武器栏固定展示：Main高亮，Offhand灰阶+被动图标。  
  - 主动技能CD环、可用闪光、资源锁红标。  
  - 主武器切换CD与退场冷却双计时。  
  - 资源锁原因文案：`冷却中/资源不足/被打断/切换锁定`。
- 涉及模块：HUD、状态同步、文案本地化。
- 对玩法影响：  
  - 收益：可预期、可计划。  
  - 代价：HUD信息密度上升。
- 风险与规避：  
  - 风险：信息过载。  
  - 规避：战斗中仅显示关键4项，详细放扩展面板。
- 验收标准：  
  - 可用性测试中，“无法判断为何未触发”反馈比例<`10%`。
- 优先级：P0  
- 预计工作量：中

## M9 触发反馈与战斗日志
- 目标问题：被动触发不可见，构筑体验不可信。
- 具体改动：  
  - 副手触发时显示图标闪烁+短文本（如“连锁就绪”“ICD中”）。  
  - 最近3条战斗日志：成功触发/失败原因/来源武器。  
  - 关键触发（处决、过热爆）加入屏幕边缘提示。
- 涉及模块：战斗事件总线、日志UI、音效。
- 对玩法影响：  
  - 收益：玩家能学习“为何强”。  
  - 代价：UI和音效资产需求增加。
- 风险与规避：  
  - 风险：提示刷屏。  
  - 规避：同类事件节流`0.5~1.0秒`。
- 验收标准：  
  - 新手30分钟后可正确描述至少2条副手触发条件。
- 优先级：P1  
- 预计工作量：小-中

## M10 数据预算与平衡框架
- 目标问题：新系统上线后容易出现单点最优解。
- 具体改动：  
  - 输出预算：Main贡献`55%~70%`；Offhand合计`30%~45%`。  
  - 主动技能预算：总输出占比`20%~30%`。  
  - 单副手上限：`<=15%`总输出。  
  - 首轮指标：切换频率、主动技能使用率、构筑多样性（Top3构筑占比）、失败原因分布。  
  - AB参数：切换CD、退场冷却、被动ICD、接力奖励幅度。
- 涉及模块：数值后台、埋点、报表、平衡流程。
- 对玩法影响：  
  - 收益：可持续调优，不靠主观感受。  
  - 代价：数据链路建设成本。
- 风险与规避：  
  - 风险：只看DPS忽视手感。  
  - 规避：指标同时纳入“输入负担”和“技能空转率”。
- 验收标准：  
  - 版本两周后，Top1构筑占比不超过`25%`，Top3不超过`60%`。
- 优先级：P1  
- 预计工作量：中

## 最小可行版本（MVP）范围

1. 必做：M1、M2、M3（简化版）、M4（仅1类被动）、M5、M8。  
2. 可延期：M6-M7（先做1类敌人试点）、M9详细日志、M10高级AB。  
3. MVP参数建议：  
   - 主手基础加成：伤害`+20%`，攻速`+15%`。  
   - 主动技能：统一CD`18秒`，资源消耗`30%`。  
   - 切换CD`2秒`，退场冷却`4秒`，全局锁`0.7秒`。  
   - 副手被动ICD默认`0.8秒`。  

4. MVP验收门槛：  
   - 攻击不再齐射（功能正确率100%）。  
   - 主手切换使用率较旧版提升`>25%`。  
   - 主动技能平均使用`>=2次/分钟`。  
   - 玩家能在UI上识别“未触发原因”准确率`>=80%`。
