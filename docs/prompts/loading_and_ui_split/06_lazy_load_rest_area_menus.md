# 提示词 06：延迟加载休整区入口菜单

请新增 `UI/scenes/runtime/rest_area_primary_menus.tscn`，将五套休整区一级入口菜单从 World 首帧移除，并在首次进入休整区时统一实例化、绑定和复用。

## 开始前

检查当前 UI 场景、休整区进入/退出流程、相关 Controller、返回按钮和本地化刷新逻辑。识别五套菜单的全部外部依赖、信号和默认可见状态。

## 实施要求

- 迁移 Purchase、Upgrade、Warehouse、Board Edit、Battle Start Primary Menu。
- 提供幂等且可重入的 `ensure_rest_area_ui()`；第一次调用实例化并统一绑定现有 Controller，后续调用复用同一实例。
- World 首帧不得创建这五套菜单或触发其重型视图初始化。
- 处理快速重复进入、退出过程中再次进入、节点被意外释放和初始化失败；不得留下半绑定状态。
- 信号只连接一次，返回路径和焦点/可见性状态每次进入都正确重置。
- 延迟创建后立即应用当前语言、主题和响应式布局状态。
- 不在本任务中卸载实例；首次创建后保留复用。

## 验收与验证

- 用节点树、日志或计数证明 World 首帧没有五套入口菜单，首次进入时只创建一次，第二次进入复用。
- 验证所有入口、返回按钮、Battle Start、语言切换和窗口尺寸变化。
- 验证直接进入休整区或异常调用顺序不会产生空引用。
- 运行 Godot `--check-only` 和完整的“World→休整区→各入口→返回→再次进入”聚焦验证。

完成后说明生命周期、Controller 绑定点、重复调用防护、首帧收益和验证结果。
