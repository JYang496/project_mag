# 提示词 03：合并 World 多线程加载实现

请把新游戏和继续游戏中重复的 World 多线程加载、轮询、进度和错误处理提取为共享组件 `World/world_scene_loader.gd`，同时保持两条流程各自的状态准备语义。

## 开始前

完整阅读 `World/new_game_btn.gd`、`World/continue.gd` 及其直接依赖，比较两条路径的共同点和差异。确认 Godot 当前使用的 `ResourceLoader.load_threaded_request()`、状态轮询和场景切换方式。

## 实施要求

- 共享加载器只负责请求并加载 World、报告进度、返回资源或错误，以及安全处理取消；不负责新游戏/继续游戏的数据差异。
- 两个按钮继续负责各自的运行状态准备、按钮文案、存档处理和最终场景切换。
- 统一处理请求失败、轮询失败、资源为空、取消、节点退出树和重复点击。
- 任何结束路径都必须恢复按钮/加载状态，不留下半初始化运行状态。
- 进度语义统一为：0%–10% 准备运行状态，10%–80% 加载 World 资源，80%–100% 构建战场。不得用虚假循环伪造进度；最后阶段应由真实 ready/build 信号推进。
- 接入 `threaded_load_started`、`threaded_load_finished`、`world_scene_changed` 等性能标记。
- 避免共享加载器持有已经释放的按钮或场景节点引用；信号连接应可控且不重复。

## 验收与验证

- 证明新游戏和继续游戏使用同一个加载实现。
- 验证正常完成、资源请求失败、取消、快速重复点击、离开菜单和再次尝试。
- 验证进度单调、不会卡在 100% 前后，场景只切换一次。
- 运行 Godot `--check-only` 并执行可行的聚焦流程验证。

完成后总结共享 API、两条入口保留的职责、故障恢复行为和实际加载时序。
