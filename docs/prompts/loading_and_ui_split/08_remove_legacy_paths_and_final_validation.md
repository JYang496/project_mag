# 提示词 08：清理旧 UI 路径并完成整体验收

请在前八个任务完成的代码上做收尾：移除已废弃的 UI 节点路径和兼容残留，完成端到端功能与性能验收。不要在本任务中引入新的大规模架构改造。

## 开始前

阅读优化计划和本目录全部提示词，检查当前工作树与此前任务总结。搜索旧 `UI.gd` 长路径、旧 bootstrap、重复 World 加载代码、常规流程中的缓存清理，以及已经迁移节点的旧引用。逐项确认是否确实废弃后再删除。

## 实施要求

- 删除已无调用方的旧节点路径、重复加载实现、废弃 bootstrap 入口和无效兼容包装；保留仍被业务或场景信号使用的接口。
- 确认 `world_ready` 只等待棋盘、玩家、相机、Battle HUD、首批可见地面，不等待未请求的休整区/管理 UI/Modal。
- 审计所有按需初始化入口的幂等性、失败恢复、信号连接和释放安全。
- 审计资源缓存边界，确保跨局复用定义但不泄漏运行时库存或可变本局状态。
- 审计本地化、响应式布局、暂停、Game Over、奖励和返回标题后的第二次进入。
- 保留必要的性能埋点；若临时诊断日志不再需要则移除或置于显式调试开关后。

## 端到端验收

- 新游戏：标题预热→加载→World→战斗→休整区→管理功能→返回。
- 继续游戏：存档准备→共享加载器→World→各 UI 功能。
- 返回标题后第二次进入：静态资源不重复扫描，延迟 UI 不重复连接或残留旧引用。
- 错误路径：预热失败、加载失败、取消/离开、快速重复点击均可恢复。
- UI：战斗 HUD、暂停、Game Over、五个休整区入口、购买、升级、仓库、棋盘、任务、奖励和所有 Modal。
- 显示：至少两种语言、窗口尺寸变化和响应式布局。
- 性能：对比提示词 00 基线，报告点击开始到 threaded load、threaded load、World 构建、world_ready、first stable frame、超过 33 ms 帧，以及第二次进入数据。

## 验证要求

- 运行 Godot `--check-only`。
- 按 `tests/README.md` 运行受影响测试选择；当前无活动测试时明确说明，并执行可行的聚焦运行验证。
- 不以“场景能启动”代替成功；检查输出中的脚本错误、空引用、重复信号和断言。
- 输出最终验收矩阵，标注通过、失败、未验证，并给出失败项的复现方式。

完成后总结最终架构、删除内容、功能结果、真实性能对比、尚未验证项和建议的后续工作。
