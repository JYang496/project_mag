RestArea Helper 拆分报告

生成日期：2026-06-25；范围：World/rest_area.gd 九宫格交互、自动导航、菜单桥接

结论

本次把 World/rest_area.gd 中低耦合的几何命中、UI 菜单桥接和玩家自动导航 API 调用拆成独立 helper。原 RestArea 仍保留公开/私有方法名作为兼容层，外部调用和现有测试不需要改入口。

新增 Helper

文件 职责
World/rest_area_zone_helper.gd 九宫格 bounds、zone rect、zone center、global point hit-test、可交互 zone 判定。
World/rest_area_menu_bridge.gd 访问 GlobalVariables.ui.rest_area_ui_controller，处理导航锁、右键取消、打开/关闭菜单、阻塞 UI 命中和 hover hint 清理。
World/rest_area_auto_navigation.gd 封装玩家 start_auto_nav、stop_auto_nav、到达检测、移动速度覆盖和中心 hold 加速。

RestArea 保留职责

  • 阶段切换、淡入淡出、奖励/路线流程。
  • 九宫格绘制、提示文本生成和 hold-to-start 进度绘制。
  • 状态字段维护，例如 selected_zone_id、hover_zone_id、menu_open、is_auto_moving。
  • 旧方法名兼容：_get_zone_rect_local()、_get_zone_id_for_global_point()、_is_mouse_over_ui() 等仍存在，但内部委托 helper。

验证结果

  • Godot --headless --path . --quit：通过。
  • World/Test/run_rest_area_helper_test_headless.gd：通过，覆盖 zone 几何和可交互分类。
  • 既有 run_rest_area_zone_hint_test_headless.gd 和 run_management_ui_polish_test_headless.gd 在当前 --script 启动方式下仍会因直接引用 autoload 全局名而编译失败；项目级编译可通过。
