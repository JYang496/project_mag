武器、分支、模组与武器联动的系统定位建议

本文基于当前项目已有设计整理：武器拥有 fuse 突破和等级上限，分支通过 WeaponBranchBehavior 改变行为，模组通过 trait、stat modifier、on-hit plugin 和被动事件接入武器。建议目标是让四层系统各自承担清晰职责，并形成从前期到后期的成长链条。

总纲： 武器提供前期伤害、手感和触发基础；武器分支决定打法路线；模组把武器行为转化为中后期主要伤害； 武器联动通过跨武器、跨模组、跨被动事件形成后期输出上限。

一、系统分层定位

系统 阶段定位 核心职责 设计边界
武器 前期核心 基础直接伤害、攻击节奏、命中事件、换弹事件、热量/弹药等基础循环。 武器可以成长，但不应独立承担中后期全部伤害；它更应该是模组和联动的触发器。
武器分支 中期转型 在 fuse 2 和 fuse 3 时提供强选择，改变攻击形态、伤害类型、范围、穿透、热量消耗或副手价值。 分支应通过行为改造带来质变，避免只做线性数值加成。
模组 中后期核心 读取武器 trait 和事件，把命中、换弹、热量、异常、移动、击杀等行为转化为额外机制和额外伤害。 模组可以强，但应依赖触发条件、兼容 trait 和等级预算，避免成为所有武器通用的无脑乘区。
武器联动 后期核心玩法 让主武器、副武器、分支、模组、被动事件互相放大，形成构筑上限。 联动可以几何倍提升总输出，但必须建立在更高构筑门槛、资源成本和可读反馈之上。

二、成长曲线建议

当前 fuse 规则已经适合承担阶段门槛：fuse 1 = level cap 3、 fuse 2 = level cap 5 + 第一次分支、 fuse 3 = level cap 7 + 第二次分支。建议把它明确设计成“成长阶段转换器”，而不是纯数值突破。

阶段 玩家主要感受 主要伤害来源 建议伤害占比 设计重点
前期 拿到武器，理解射击、命中、换弹、热量和范围。 武器直接伤害 约 70% 到 90% 武器手感必须可靠；模组只做轻量辅助或明确触发演示。
中期 通过重复武器突破，第一次选择分支，开始围绕 trait 和触发条件选模组。 分支 + 模组 约 50% 到 70% 分支改变打法，模组开始接管输出成长。
后期 第二分支成型，多武器之间有明确分工，模组链条开始互相放大。 模组 + 武器联动 约 60% 到 85% 允许高上限组合，但用触发条件、冷却、资源和 UI 反馈控制可读性。

三、各系统具体建议

武器：基础行为和触发接口

直接伤害 手感 触发器

武器的直接伤害应保证前期能独立推进战斗，但中后期的价值应逐步转向“高质量触发源”： 命中频率、伤害类型、弹药消耗、换弹节奏、热量消耗、主副手状态，都会决定它能触发哪些模组和联动。

  • 每把武器至少要有一个清晰的触发标签，例如 projectile、melee、heat、freeze、area_of_effect。
  • 武器升级主要改善基础体验：伤害、冷却、弹药、范围、投射物速度。
  • 避免让武器等级单独提供过高乘法成长，否则分支和模组会失去构筑价值。

武器分支：路线和质变

打法路线 形态变化 中期选择

分支应回答“这把武器接下来要成为哪种构筑核心”。当前的分支行为接口已经支持弹道、伤害类型、 冷却、穿透、热量、爆炸配置和被动事件，适合承载质变。

  • 每个分支优先改变行为，再附带少量数值。
  • 同一武器的两个分支池建议覆盖不同路线：群体、单体、异常、热量、保护、副手辅助。
  • 使用 exclusive_groups 和 incompatible_branch_ids 防止破坏性组合。

模组：中后期伤害主体

触发收益 trait 兼容 机制扩展

模组应把武器行为翻译成额外收益。纯数值模组适合补基础短板，触发型模组适合提供中后期主要伤害。 当前 Module 已经提供 stat modifier、on-hit plugin、等级成长和 trait 限制，建议继续沿用。

  • 通用数值模组预算低，专用触发模组预算高。
  • 强模组必须绑定明确条件，例如 freeze、heat、reload、projectile、melee。
  • 模组等级 1 到 3 应改变“可靠性或覆盖率”，不要只堆伤害倍率。

武器联动：后期乘区和构筑上限

跨武器 被动事件 几何成长

联动的目标不是让每把武器都独立变强，而是让玩家通过组合获得更高输出。例如一个武器负责制造状态， 另一个武器消耗状态；一个武器通过 reload 事件强化副手，另一个武器通过高频命中兑现强化。

  • 联动应优先使用现有全局被动事件和模组插件体系，减少武器之间的硬编码引用。
  • 后期允许乘法收益，但必须让 UI 显示来源、触发条件和当前状态。
  • 强联动应至少占用两个条件：指定 trait、指定事件、主副手位置、冷却、热量或弹药窗口。

四、伤害预算规则

收益类型 建议预算 适合放在哪 风险
直接加伤、减冷却、加弹药 低到中 武器等级、基础模组、小分支附带数值 过高会稀释构筑选择。
改变攻击形态 中到高 武器分支 容易同时提升清怪、单体和触发频率，需要保留短板。
命中/异常/换弹触发额外伤害 中到高 触发型模组、武器被动 高频武器会放大收益，需要内部冷却或衰减。
跨武器增益 高 后期模组、第二分支、特殊被动 乘法叠加可能失控，必须限制持续时间、条件或覆盖率。
状态制造 + 状态消费 高到极高 后期联动核心 如果状态覆盖率过高，会变成无条件全局增伤。

推荐公式思路

总输出可以被拆成：武器基础 DPS × 分支形态系数 × 模组触发收益 × 联动覆盖率。 其中联动不要直接设计成永久倍率，而应设计成“高倍率 × 有限覆盖率”。这样玩家能打出爆发窗口， 但常态输出不会被永久乘区压垮。

五、内容设计准则

1. 每把武器先定义触发身份

武器身份 适合的模组 适合的联动
高频命中 低伤害 on-hit、叠层、短 ICD 触发 消费标记、刷新短增益、触发连锁
低频爆发 高倍率、暴击、范围爆炸、击杀触发 吃前置状态、消耗热量、爆发窗口
异常制造 冻结、燃烧、腐蚀、减速、易伤 给其他武器提供消费对象
换弹/热量循环 reload、heat、overheat、ammo spend 强化副手、触发全局事件、窗口爆发
近战/轨迹覆盖 trail、contact、area、lifesteal、shield 控制战场位置，为远程武器创造输出窗口

2. 分支要保留路线短板

  • 推荐：长射程分支提升安全性，但降低清群或冷却；范围分支提升覆盖，但降低单体。
  • 推荐：热量分支提高爆发，但增加热量管理压力。
  • 谨慎：同时增加投射物数量、基础伤害、触发频率和范围，这会直接压过模组和联动。

3. 模组按触发成本定强度

  • 无条件 stat 模组：低预算，主要补体验。
  • 单武器条件模组：中预算，例如 projectile、freeze、heat、melee 限定。
  • 需要特定事件的模组：高预算，例如换弹、击杀、连续移动、蓄力命中。
  • 需要另一把武器配合的模组：最高预算，但应占用后期资源或稀有度。

4. 联动链条必须可读

如果玩家无法判断是哪把武器、哪个分支、哪个模组在放大输出，后期构筑会变成黑箱。 建议每个强联动都能在 UI 或日志里显示“来源、目标、触发条件、持续时间、当前层数”。

六、与现有实现的对齐建议

现有能力 建议用法 下一步
FUSE_LEVEL_CAPS 与自动 fuse 作为阶段门槛，控制等级上限和分支选择。 在武器详情、奖励和商店中明确展示“突破后解锁分支/等级上限”。
WeaponBranchBehavior 承载分支质变，避免替换整把武器场景。 为每个分支补一行“路线标签”，例如单体、清群、异常、热量、副手。
Module.required_weapon_traits 把强模组限制到合适武器，防止无条件泛用。 检查现有强模组是否都绑定了足够具体的 trait。
passive_triggered 与全局事件 作为武器联动的主要通道。 统一 detail 字段，让 UI 和调试能读出联动来源。
reload/offhand 类模组 作为第一批武器联动模板。 把“主武器触发，副武器兑现”的组合做成后期推荐构筑。

七、推荐落地顺序

  1. 先补定位标签： 给武器、分支、模组增加设计侧标签，至少包括输出方式、触发方式、伤害类型、联动角色。
  2. 再补 UI 可读性： 武器详情展示当前 fuse、等级上限、分支、已装备模组、被动状态和可触发联动。
  3. 整理模组预算： 把无条件数值模组、单武器触发模组、跨武器联动模组分成不同强度档。
  4. 建立后期联动样板： 每个元素或 trait 至少设计一条“制造者 + 消费者 + 放大器”的三段链条。
  5. 用 DPS 测试验证： 针对单武器、分支成型、模组成型、双武器联动四种状态分别跑基准，确认成长曲线不是断崖或爆炸。

八、设计红线

  • 不要让武器基础等级成为最大的后期乘区，否则分支和模组只是装饰。
  • 不要让强模组无 trait 限制地适配所有武器。
  • 不要把分支做成纯数值升级，分支应该改变玩法路线。
  • 不要让武器联动永久满覆盖，后期强度应来自窗口、链条和构筑理解。
  • 不要隐藏后期输出来源，至少在调试和武器详情中能追踪触发链条。

结论

当前项目的技术结构已经适合支撑这套定位：fuse 控制阶段，分支提供质变，模组读取 trait 和事件， 被动事件承载跨武器联动。后续设计应避免把所有成长都压到武器本体上，而是让玩家逐步经历 “武器能打”到“分支定向”到“模组成型”再到“联动爆发”的完整构筑过程。
