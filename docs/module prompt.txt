你是 Godot 4/GDScript 开发者。请基于现有项目武器模组风格，设计并实现以下“新武器模组”。
- 投射物经过的路径会产生AOE，存在期间持续造成冰属性伤害，持续X秒。
- 换弹时，根据已经打出的子弹百分比对玩家周围的敌人造成一次伤害。
- 换弹时，根据已经打出的子弹百分比对玩家周围的敌人造成一次击退。
- 换弹时，根据已经打出的子弹百分比提高该武器的伤害，持续X秒。
- 换弹时，根据已经打出的子弹百分比提升玩家的移动速度，持续X秒。
- 换弹时，根据已经打出的子弹百分比给予玩家临时护盾，持续X秒。
- 换弹时，根据已经打出的子弹百分比对玩家其他武器的伤害，持续X秒。
- 换弹时，如果存在其它正在换弹的武器，本武器换弹速度提升X%。

【项目约束（必须遵守）】
1. 新模组继承 `Module`（`Player/Weapons/Modules/wmod_base.gd`）。
2. 文件命名：
   - 脚本：`Player/Weapons/Modules/wmod_<effect>_<tag>.gd`
   - 场景：`Player/Weapons/Modules/wmod_<effect>_<tag>.tscn`
3. 场景结构必须复用 `wmod_base.tscn`，并设置：
   - `script`
   - `Sprite.texture`（并且默认不可视）
   - 必要时设置 `required_weapon_traits` / `module_traits` / `supports_melee` / `supports_ranged`
4. 若是纯数值模组：优先实现 `configure_stat_modifiers()`，通过 `stat_multipliers/stat_additives` 修改属性。
5. 若是命中触发模组：实现 `apply_on_hit(source_weapon, target)`，并在 `_enter_tree/_ready` 注册 `register_as_on_hit_plugin()`，在 `_exit_tree` 反注册。
6. 所有逻辑都要做空值与接口检查（`is_instance_valid`、`has_method`）。
7. 等级成长必须兼容 `module_level`（1~3），优先使用 `get_effective_multiplier/get_effective_additive` 或 `lv1/lv2/lv3` 参数。
8. 不要破坏现有接口和模块兼容性文案格式（如 `Requires one of: ...`）。

- 模组名称：{{MODULE_NAME}}
