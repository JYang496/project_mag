你在 Windows 本地 Godot 项目 `D:\Godot Projects\project_mag` 工作。

阶段 1：补全历史 scene manifest。

目标：把 `tests/scenes/**.tscn` 中仍未登记的活跃自动化 scene 分批加入 `tests/infrastructure/test_manifest.json`，让 selected-test 机制从 pilot 代表集走向可信 full。

开始前必须执行：

```powershell
git status --short --branch
git rev-parse HEAD
```

开始前阅读：

- `tests/README.md`
- `tests/infrastructure/test_manifest.json`
- `tests/infrastructure/source_domain_map.json`
- `docs/plans/project_slimming/test_infrastructure_progress.md`
- `docs/plans/project_slimming/overall_progress.md`

不要修改业务代码。不要删除、移动或重写测试。只登记现有活跃 scene-backed 自动化测试，除非某个 runner-only 测试必须先补 scene wrapper；这种情况单独报告。

## 执行

1. 生成未登记清单：

```powershell
rg --files tests/scenes -g "*.tscn"
```

对比 `tests/infrastructure/test_manifest.json` 中已有 `path`。

2. 按 domain 分批登记，不一次性全塞：
   - UI
   - Player
   - Weapon
   - World
   - Reward
   - Cell
   - Startup
   - Spawn
   - Enemy
   - Combat

3. 每个条目必须设置：
   - `id`
   - `entry_type: "scene"`
   - `path`
   - `domain`
   - `dependency_domains`
   - `parallel_safe`
   - `writes_user_data`
   - `timeout_seconds`

4. 不确定依赖时保守处理：
   - `parallel_safe: false`
   - `writes_user_data: true`
   - `dependency_domains` 宁可多写，不要少写。

5. 每批验证：

```powershell
(Get-Content -Encoding UTF8 -Raw -Path tests/infrastructure/test_manifest.json | ConvertFrom-Json).tests.Count
powershell -NoProfile -ExecutionPolicy Bypass -File tests/infrastructure/tests/test_selection_test.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File tests/infrastructure/tests/test_worker_test.ps1 -GodotPath 'D:\SteamLibrary\steamapps\common\Godot Engine\godot.windows.opt.tools.64.exe'
powershell -NoProfile -ExecutionPolicy Bypass -File tests/infrastructure/run_test_workers.ps1 -TestId '<new_id_1>','<new_id_2>' -GodotPath 'D:\SteamLibrary\steamapps\common\Godot Engine\godot.windows.opt.tools.64.exe' -Jobs 2 -OutputRoot 'test-results\manifest-registration-<domain>'
git diff --check
```

6. 更新：
   - `tests/README.md`
   - `docs/plans/project_slimming/test_infrastructure_progress.md`
   - `docs/plans/project_slimming/overall_progress.md`
