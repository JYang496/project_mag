你负责 `project_mag` 瘦身计划的安全资源管线范围。

目标：在没有可见质量变化和路径破坏的前提下，减少冷导入负担、重复资源和运行时扫描噪音。

开始前阅读：
- `docs/prompt/project_slimming_incremental_execution_prompt.txt`
- `docs/prompt/project_slimming_parallel/00_parallel_coordinator_prompt.txt`
- `project.godot`
- `asset/**` 的引用和 `.import` 配置

开始前执行：

```powershell
git status --short --branch
git rev-parse HEAD
```

## 允许修改

- `asset/**`
- 资源引用审计报告
- 经审计批准的导入配置
- `docs/plans/project_slimming/asset_pipeline_progress.md`
- `docs/plans/project_slimming/handoffs/asset_pipeline.md`

## 禁止修改

- `project.godot`
- `UI/**`
- `Player/**`
- `autoload/**`
- `World/**`
- 测试基础设施
- 其他范围进度文件

业务代码引用变化、项目设置变化和路径迁移只写入 handoff，由协调者安排后续集成。

## 第一批只做审计

输出：
- 文件大小与类型分布
- 直接文本引用
- 场景/资源依赖引用
- `.import` 设置差异
- 疑似重复文件
- 制作源文件与运行时资源分类
- 删除或归档风险
- 建议恢复方式

重点候选：
- `asset/fonts/NotoSansSC-Regular.ttf`
- `asset/fonts/NotoSansCJKsc-Regular.otf`
- `asset/images/characters/characters_move.zip`
- 大尺寸 Cell 图片
- 重复命名或内容相同的 PNG/SVG

## 后续允许执行

仅在审计证据充分、协调者批准后：
- 统一同类资源导入预设。
- 移出不应被 Godot 扫描的制作源文件。
- 归档不参与运行时的 ZIP、源图或中间产物。
- 去除确认未引用且可恢复的重复资源。

每次只处理一类资源，并记录：
- 修改前后文件数
- 修改前后源文件体积
- 冷导入耗时
- 导入缓存体积
- 引用验证
- 视觉或音频验证

## 禁止事项

- 不降分辨率。
- 不做有损压缩。
- 不减少字体字符覆盖。
- 不降低音频质量。
- 不批量改资源路径。
- 不在没有引用证据时删除文件。
- 不把仓库体积下降当成冷导入提速证据。

注意：
- ZIP 归档可能增加仓库体积，但未必参与 Godot 导入。必须区分仓库瘦身和导入提速。
- 字体文件体积大不等于重复。必须检查字符覆盖、Fallback 和本地化需求。

## 验证

- 资源引用扫描。
- Godot 语法/资源检查。
- 启动场景和相关 UI 场景加载。
- 中英文字体覆盖检查。
- 涉及图片时做修改前后视觉对比。
- 冷导入基线对比。
- `git diff --check`。

## 停止条件

- 无法证明资源未使用。
- 处理会改变视觉、字体覆盖或音频质量。
- 需要修改业务代码或 `project.godot`。
- 冷导入没有可测收益。
- 恢复路径不明确。
