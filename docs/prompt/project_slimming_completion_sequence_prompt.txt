你在 Windows 本地 Godot 项目 `D:\Godot Projects\project_mag` 工作。

任务：按既定顺序完成项目瘦身计划的剩余任务。当前目标不是采用 `ponytail` 的进一步裁剪建议，而是先把瘦身计划收口到可验证、可维护、可交接的状态。`ponytail` 只在全部阶段完成后作为复盘优化建议使用。

本文件是阶段索引。执行时每次只打开并执行一个阶段文件，不要跨阶段混批。

## 开始前通用要求

每个阶段开始前都必须执行：

```powershell
git status --short --branch
git rev-parse HEAD
```

不要回滚或覆盖其他人的改动。只修改当前阶段需要的文件。若目标文件存在无法安全合并的并行改动，停止并报告冲突。

## 执行规则

- 当前阶段失败时停止，不顺带进入下一阶段。
- 每阶段结束必须更新对应 progress 文件和 `docs/plans/project_slimming/overall_progress.md`。
- 高风险职责迁移前必须先补 focused scene-backed gate。
- 不为了减少行数裸拆 `UI.gd` 或 `Player.gd`。
- 不新增 Autoload、服务定位器、万能 Context 或每帧临时容器。
- 不删除历史测试、资源或工具，除非该阶段明确包含审计、恢复方案和验证。
