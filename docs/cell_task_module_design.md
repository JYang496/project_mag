# 格子任务模组设计合同

## 目标

把当前格子系统扩展为“地形格子 + 槽位 + 任务模组 + 地形奖励”的闭环：

1. 玩家在准备阶段通过一级菜单进入格子管理。
2. 任务管理页部署短期任务模组到下一场战斗的 active cell。
3. 战斗中完成格子任务后，成功结算时解锁奖励选择。
4. 奖励选择以地形效果为主，也可能提供新的任务模组。
5. 地形效果进入未安装库存，再由格子管理页安装到格子。

第一版目标是正式可玩闭环，不是只做技术原型。

## 已锁定规则

### 菜单与信息架构

- 新增一级菜单：`格子管理`。
- 点击一级菜单后先进入两个并列入口：
  - `格子管理`
  - `任务管理`
- `格子管理`页负责地形效果：
  - 查看 3x3 格子的地形效果。
  - 查看 active cell 状态。
  - 安装未安装地形效果。
  - 替换已安装地形效果。
- `任务管理`页负责任务模组：
  - 查看短期任务模组库存。
  - 部署任务模组到 active cell。
  - 任务模组库存卡只展示名称、任务类型和稀有度颜色，不展示详细描述。
  - 鼠标悬停左侧已安装任务的 cell 时，固定详情窗口临时展示该 cell 的任务详情。
  - 点击左侧已安装任务的 cell 时，详情窗口常驻；继续悬停其他已安装任务 cell 会临时覆盖，移开后恢复点击锁定的 cell。
  - 取消未开战部署。
  - Ready To Install 库存不设数量上限。

### 格子槽位

- 第一版每个格子只有 1 个任务槽。
- 每场战斗最多安装/激活 2 个任务模组。
- 任务模组只能安装到下一场战斗的 active cells。
- 所有 active cell 都能装任意任务模组，不做地形兼容限制。
- 任务和地形光环第一版互不影响。

### 任务模组定位

- 任务模组是局内物品，但使用独立库存。
- 不复用武器模组或临时模组库存。
- 任务模组只代表任务大类，具体任务参数由格子、关卡或当前平衡配置自动生成。
- 第一版开放 5 类基础任务：
  - 击杀
  - 守点
  - 清场
  - 猎杀精英
  - 闪避生存
- 任务模组有稀有度，但稀有度只影响出现权重。
- 稀有度不改变任务行为、任务难度或奖励质量。
- 所有稀有度都能生成 5 类任务，只是权重轻微不同。

### 任务模组生命周期

- 任务模组库存只保留到下一次准备阶段。
- 玩家获得任务模组后，必须在下一次准备阶段部署、替换或放弃。
- 战斗开始前可以取消部署，取消会退回任务模组库存。
- 进入战斗后，已部署任务模组正式消耗。
- 进入战斗时如果还有未部署任务模组：
  - 弹确认提示。
  - 玩家确认后清空未部署任务模组并进入战斗。
- 已部署但未完成的任务模组也会消耗。
- 战斗结束后任务槽位清空。
- 死亡或失败时，本场所有格子任务奖励作废。

### 单场任务完成

- 每场战斗可以完成多个格子任务。
- 每个格子任务只能完成一次。
- 因为每场最多激活 2 个任务模组，所以单场最多产生 2 次格子任务奖励机会。

### 奖励池

- 完成格子任务后，成功进入准备阶段时触发奖励选择。
- 奖励固定为 3 选 1。
- 每次奖励至少 2 个选项是地形效果。
- 每次奖励最多 1 个选项是任务模组。
- 奖励里的任务模组显示确定的任务类型，稀有度仅通过颜色表达，不显示稀有度文字。
- 玩家选择新的任务模组后直接加入 Ready To Install 库存。

### 地形效果

- 完成任务后获得的是地形效果到未安装库存或待安装池。
- 不自动安装到完成任务的格子。
- 不在当前战斗中立刻改变地形。
- 玩家在准备阶段通过格子管理页安装地形效果。
- 已安装地形不能拆回库存。
- 已安装地形只能被新地形替换。
- 替换会销毁旧地形。
- 替换已安装地形必须确认，并明确提示旧地形会被销毁。

### 获取方式

- 任务模组只通过完成格子任务后的奖励选择获得。
- 不提供商店、兑换或未安装地形效果换取任务模组的入口。

### 战斗内显示与结算

- 格子上显示任务图标和任务进度。
- HUD 只显示汇总信息：
  - 本场激活任务数。
  - 已完成任务数。
  - 是否有奖励待领取。
- 战斗结算需要提示：
  - 已完成任务。
  - 未完成任务。
  - 已消耗任务模组。
  - 获得的奖励机会。

## 实施拆分

### 阶段 1：数据与运行时状态

- 添加 `TaskModuleDefinition` 资源脚本。
- 添加 5 个基础任务模组资源。
- 添加任务模组短期库存和部署运行时。
- 支持 Ready To Install 库存不设数量上限。
- 支持准备阶段部署和取消部署。
- 支持进入战斗时提交消耗。
- 支持未部署任务模组清空确认的数据接口。

### 阶段 2：格子任务应用

- 在 active cell 上应用已部署任务模组。
- 根据任务类型启用对应 `CellObjectiveModule`。
- 任务参数仍来自当前关卡、格子或配置，不从任务模组读取。
- 支持每格任务只完成一次。
- 支持战斗结束后清空任务槽。
- 支持失败时清除本场任务奖励。

### 阶段 3：奖励池扩展

- 扩展奖励类型，支持任务模组选项。
- 保持 3 选 1。
- 保证至少 2 个地形效果。
- 保证最多 1 个任务模组。
- 奖励任务模组显示确定类型，稀有度仅通过颜色表达。
- 奖励任务模组直接加入 Ready To Install 库存。

### 阶段 4：格子管理一级菜单

- 新增一级 `格子管理` 菜单入口。
- 首屏展示 `格子管理` 和 `任务管理` 两个入口。
- `格子管理`页实现：
  - 3x3 格子状态。
  - active cell 标记。
  - 已安装地形显示。
  - 未安装地形库存。
  - 安装和替换确认。
- `任务管理`页实现：
  - 短期任务模组库存。
  - 先选任务模组，再选目标 active cell。
  - 已部署任务显示。
  - 已部署任务详情通过 cell hover / click 弹窗查看。
  - 取消部署。

### 阶段 5：战斗显示与结算

- 格子上显示任务图标和进度。
- HUD 添加任务汇总。
- 战斗成功结算显示完成、未完成和消耗。
- 死亡/失败时明确不保留本场任务奖励。

## 验收清单

### 数据

- 5 类基础任务模组都能被读取。
- 每个任务模组都有确定任务类型和稀有度；稀有度在任务管理中只用颜色表示。
- 稀有度只影响生成权重，不影响任务参数。

### 库存与部署

- Ready To Install 任务模组库存不设数量上限。
- 领取新任务模组会直接加入 Ready To Install 库存。
- 准备阶段可以部署到下一战 active cell。
- 不能部署到非 active cell。
- 战斗开始前取消部署会退回库存。
- 进入战斗后已部署任务模组被消耗。
- 进入战斗时未部署任务模组会触发确认并清空。

### 战斗任务

- 单场最多激活 2 个格子任务。
- 每个格子任务只完成一次。
- 多个格子任务可以在同一场战斗中分别完成。
- 未完成任务在战斗结束后也被消耗并清空。
- 失败或死亡会清除本场所有格子任务奖励。

### 奖励

- 每个完成任务在成功进入准备阶段后生成 3 选 1。
- 每组奖励至少 2 个地形效果。
- 每组奖励最多 1 个任务模组。
- 任务模组选项显示确定类型，稀有度仅通过颜色表达。
- 获得的地形效果进入未安装库存。
- 地形效果不会自动安装到完成任务的格子。

### 格子管理

- 一级菜单可以进入格子管理系统。
- 首屏有 `格子管理` 和 `任务管理` 两个入口。
- 格子管理页能安装未安装地形。
- 替换已安装地形时必须确认。
- 被替换的旧地形会销毁，不能回库存。
- 任务管理页库存卡片不显示任务描述。
- 任务管理页左侧已安装任务 cell 支持 hover 详情预览。
- 点击已安装任务 cell 后详情窗口常驻，Esc/取消键或底部 Close 都能关闭。
